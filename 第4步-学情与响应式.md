## 响应式

- 仍以 **&lt;768px = 家长**、**≥768px = 孩子** 为准（`AppContext` + `matchMedia`）。
- `html[data-mode="parent" | "child"]` 在 `RootLayout` 中设置，用于**根字体**等差别（`index.css`）。
- 孩子端底栏**更大**触摸目标、图标、主导航；页头徽标在宽屏上略放大。主内容区随模式调整内边与最小高度。

## 课标与「客观」学情：为何没有调教育局 API

- 目前**国家/地方教育行政机构**对公众商业 App **几乎不提供**可直连的、统一的「学情/知识库」开放 HTTP API；课程标准多以**公开 PDF/网页**存在（如 2022 年版义务教育**课程标准**），需自行**结构化、维护词表**。
- 本仓库的折中实现：**`src/data/curriculumReference.sample.json`** 将课标中常见**能力域 + 关键词**做示例；**`src/utils/curriculumMatch.js`** 用错题的**题干+知识点**与词表**匹配**并聚合成「课标能力域分布」块（见 `CurriculumAlignmentSection`）。这基于**可审计的课标语汇**，但仍是**规则/关键词层面**的自动归类，**不是**统考级测评。
- 若要进一步「更客观/更贴课标说明」，可：扩充 JSON、由教研在后台维护映射表，或对课标原文做 **RAG + 大模型** 生成解释；仍通常走**自研/采购后端**，而不是调用不存在的「教育局知识库 API」。
