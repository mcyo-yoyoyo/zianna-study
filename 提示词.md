# 角色设定
你现在是一个全栈开发工程师、资深AI产品经理，以及精通儿童UI/UX的设计师。请帮我开发一款名为「伴学精灵」的跨端H5教育应用，核心目标用户为小学一年级学生及其家长。

# 产品愿景与定位
一款专为一年级学生设计的AI辅助学习Web App。解决孩子独立完成听写、错题整理的痛点，解放家长。产品需要同时适配Pad端（孩子使用，横屏/大字号/卡通趣味化）和手机端（家长使用，竖屏/数据看板/高效管理）。

# 技术栈建议
- 前端框架：React / Next.js 或 Vue3 (按你最擅长的生成)
- 样式：Tailwind CSS (便于快速响应式布局)
- 后端/数据存储：LeanCloud 或 Supabase (用于多端数据实时同步和身份验证)
- AI接口：接入支持Vision API的大模型（如OpenAI GPT-4o或Anthropic Claude 3.5 Sonnet）替代传统OCR；接入Web Speech API实现TTS语音播报。

# 核心功能模块与业务逻辑

## 模块一：AI智能听写 (解决家长无法陪读的痛点)
1. **家长端（手机端）：任务下发**
   - 拍照/相册上传功能：支持上传钉钉群里的田字格听写图片。
   - **核心技术点（Vision LLM）**：将图片发送至多模态大模型，Prompt设定为：“识别图片中的田字格生字/词语，忽略田字格边框线和背景干扰，以JSON数组格式输出需要听写的词汇列表”。
   - 词汇核对与配置：展示AI提取的词汇列表，家长可手动增删改。设置每个词的播报间隔（如5秒、10秒，支持自定义）、重复次数（如读2遍）。
   - 一键下发至孩子端。
2. **孩子端（Pad端）：独立听写**
   - 界面设计：大按钮，游戏化UI（如点击“开始挑战”，一只虚拟宠物伴随）。
   - 语音交互：使用浏览器TTS自动播报，屏幕提供清晰的进度提示（如1/10）。提供“上一词”、“重读”、“下一词”、“暂停”的大图标控制键。
   - 听写结束：孩子用Pad拍照上传听写本，系统通过AI对比正确答案，圈出可能错的字（MVP阶段可由家长人工批改）。

## 模块二：AI举一反三错题本 (解决知识盲点痛点)
1. **拍照与智能切题**：
   - 支持连拍或单拍试卷/练习册。
   - AI自动框选错题，进行图像预处理（去手写痕迹、去黑边）。
2. **知识点图谱映射与归档**：
   - 结合大模型，识别题目所属学科（语/数/英），并打上标签（如：深圳部编版一年级下册、知识点：100以内的加减法/拼音轻声）。
   - 自动归档至对应学科的错题集。
3. **“消灭错题”挑战（孩子端）**：
   - 趣味练习：基于错题记录，调用LLM自动生成3道同等难度的相似题（举一反三）。
   - 闯关模式：答对后获得星星奖励，错题移出“易错池”。
4. **学情看板（家长端）**：
   - 雷达图展示知识点掌握情况。
   - 总结高频错误点，并基于内置的“深圳地区高频考点库”给出复习建议。

# 交互与UI/UX规范 (重点关注)
1. **适龄化设计**：Pad端不要有复杂的表单，少用纯文字，多用Icon和语音提示。配色使用明快活泼的马卡龙色系。
2. **响应式设计**：必须严格区分 `viewport`。手机端（宽度<768px）展示家长视图；Pad端（宽度>=768px）默认展示孩子视图。
3. **数据互通**：确保基于同一个账号登录，手机端上传的任务，Pad端能无延迟刷新出来。
