# 第三步：错题本 + 知识点标签

## 已具备能力

1. **家长机（&lt;768px）**  
   - `PhotoUpload`：选图 → 调多模态 API 用 JSON 抽错题（`src/utils/llmErrorBook.js` 的 `extractErrorsFromPhoto`）→ 写入 `localStorage`；不识别时可 **手录**。  
   - `KnowledgeGraph`：按语/数/英 分栏，**知识点**气泡与计数；**深圳**方向复习短句为内置 `shenzhenHints.js`；可删/改易错、切换是否在易错池。  

2. **孩子机（≥768px）**  
   - `PracticeChallenge`：在 **易错池** 选一条 → 调 **仅文本**（`generateSimilarQuestions`，可用 `VITE_LLM_TEXT_MODEL`）出 3 道相近题 → 闯关；最后一题都「会了」则把本题移出易错池并加 ★。  

3. **数据**  
   - 键名见 `errorBookStore.js`（`banxue_errorbook_v1`），与听写本一样暂存**本机**，第四步再接云。  

## 环境

- 识图与出同类题**共用** `.env` 里的 `VITE_DEEPSEEK_API_KEY` / `VITE_OPENAI_BASE_URL` 等。  
- 若模型仅支持文本：`extractErrorsFromPhoto` 会失败 → 用 **手录**；`generateSimilarQuestions` 在 DeepSeek `deepseek-chat` 上通常可正常用。  
- 可选 `VITE_LLM_TEXT_MODEL=deepseek-chat` 专门指定**文字题**（与多模态模型可分开，仅当你两种模型不同时需填）。  

## 与「提示词」的对应

- 拍照 / 向 LLM 要题面+学科+知识点+册别：已实现（识图手录两路）。  
- 去手写/裁边：未做图像后处理，文档里已说明可后接。  
- 学情看板/雷达图：在第四步或 Dashboard 扩展，本步仅家长侧列表与「深圳」短提示。
