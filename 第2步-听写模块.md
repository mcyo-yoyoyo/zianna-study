# 第二步：AI 听写（Vision + TTS）

## 功能

- **家长端（&lt;768px）**：上传田字格听写表 → 调 OpenAI 兼容 **Vision** 接口抽词（`src/utils/visionExtract.js`）→ 可增删改词、设词间停顿与每词朗读遍数 → **下发到本机**（`localStorage`）。
- **孩子端（≥768px）**：`Web Speech API` 自动朗读（`src/hooks/useSpellingPlayer.js`）+ 大按钮：开始/暂停、继续、上一词/下一词/重读；结束页可本地选图（不上传，MVP）。

## 环境

1. 复制 `.env.example` 为根目录 `.env`。
2. 填写 `VITE_OPENAI_API_KEY`（或仅在家长端页面「临时密钥」中填写，**勿用于公网无防护环境**）。

```text
VITE_OPENAI_API_KEY=sk-...
# 可选：代理/兼容基座
# VITE_OPENAI_BASE_URL=https://api.openai.com/v1
# VITE_OPENAI_VISION_MODEL=gpt-4o
```

3. `npm run dev` 后：窄窗作家长、拉宽到 ≥768px 作孩子；同浏览器里「下发」后，孩子端会刷出新任务（`storage` + 自定义事件）。

## 多设备

当前仅**本机** `localStorage`。手机 + Pad 两端实时同步在 **第四步** 接 LeanCloud / Supabase 后实现。

## 技术说明

- TTS 使用 `speechSynthesis` + `zh-CN`；各浏览器语音质量略有差异，建议在 **Chrome/Edge** 上试。
- 正式环境请**不要**把 OpenAI 密钥写进前端，应通过云函数/自建 API 代调多模态模型。
- **DeepSeek** 的 `deepseek-chat` 为**纯文本** API，**不支持**在 `messages` 里发 `image_url`（会报 `unknown variant image_url`）。本仓库通过 `supportsChatVision()` 对 `api.deepseek.com` 等自动**不发图**；识图需换多模态模型或换能转发 Vision 的代理，并可用 `VITE_LLM_SUPPORTS_VISION=true` 显式开启。否则用「加一行」手输听写词即可。
